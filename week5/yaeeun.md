# 5.1. 복잡도

## 5.1.1. 시간 복잡도

---

### 빅오 표기법

: 시간 복잡도를 표기하는 법.

- 입력 범위 n을 기준으로 로직이 몇 번 반복되는지 나타내는 것
- 가장 큰 차수를 기준으로 작성. 나머지 차수와 계수는 무시.

### 시간 복잡도

: 입력 크기에 대해 어떠한 알고리즘이 실행되는 데 걸리는 시간

- 로직의 반복 횟수를 중점으로 측정

## 5.1.2. 공간 복잡도

---

### 공간 복잡도

: 프로그램을 실행시켰을 때 필요로 하는 자원의 양

## 5.1.3. 자료 구조에서의 시간 복잡도

---

- 평균 & 최악의 시간 복잡도를 고려해 사용

| --- | --- | --- | --- | --- |
| 배열(array) | O(1) | O(n) | O(n) | O(n) |
| 스택(stack) | O(n) | O(n) | O(1) | O(1) |
| 큐(queue) | O(n) | O(n) | O(1) | O(1) |
| 이중 연결 리스트(doubly linked list) | O(n) | O(n) | O(1) | O(1) |
| 해시 테이블(hash table) | O(1) | O(1) | O(1) | O(1) |
| 이진 탐색 트리(BST) | O(logn) | O(logn) | O(logn) | O(logn) |
| AVL 트리 | O(logn) | O(logn) | O(logn) | O(logn) |
| 레드 블랙 트리 | O(logn) | O(logn) | O(logn) | O(logn) |

# 5.2. 선형 자료 구조

: 요소가 일렬로 나열되어 있는 자료 구조

## 5.2.1. 연결 리스트

---

: 데이터를 감싼 노드를 포인터로 연결한 자료 구조

### 종류

1. 싱글 연결 리스트 : next 포인터만 있음. → 일방향
2. 이중 연결 리스트 : next 포인터와 prev 포인터를 가짐. → 양방향
3. 원형 이중 연결 리스트 : 이중 연결 리스트에서 마지막 노드의 next 포인터가 헤드 노드를 가리킴.

- 헤드(head) : 맨 앞에 있는 노드

## 5.2.2. 배열

---

: 같은 타입의 변수들로 이루어져 있고, 크기가 정해진, 인접한 메모리 위치에 있는 데이터를 모아놓은 집합

- 중복 허용. 순서 있음

### 랜덤 접근과 순차적 접근

- 랜덤 접근 : 직접 접근. 임의의 인덱스에 해당하는 데이터에 접근할 수 있는 기능
- 순차적 접근 : 데이터를 저장된 순서대로 검색해야 함.

- 배열은 랜덤 접근 가능
- 연결 리스트는 순차적 접근

## 5.2.3. 벡터

---

: 동적으로 요소를 할당할 수 있는 동적 배열

- 컴파일 시점에 개수를 모를 때 사용
- 중복 허용, 순서가 있고 랜덤 접근 가능

## 5.2.4. 스택

---

: LIFO(마지막으로 들어간 데이터가 가장 첫 번째로 나옴) 성질을 가진 자료구조

- 재귀적 함수, 알고리즘, 웹 브라우저 방문 기록에 사용

## 5.2.5. 큐

---

: FIFO(먼저 집어넣은 데이터가 먼저 나옴) 성질을 가진 자료구조

- CPU 작업을 기다리는 프로세스, 스레드 행렬, 네트워크 접속을 기다리는 행렬, 너비 우선 탐색, 캐시 등에 사용.
