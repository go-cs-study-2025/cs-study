비선형 자료 구조란 자료 순서나 관계가 복잡한 구조로 트리, 그래프 등이 있다

## 1. 그래프

- 그래프는 정점(vertex)과 간선(edge)로 이루어져 있다
- 그래프의 가중치란 간선과 정점 사이에 드는 비용

## 2. 트리

- 트리는 그래프 중 하나로 그래프처럼 정점, 간선으로 이루어져 있다
- 루트 노드, 내부 노드, 리프 노드 등으로 구성된다
    - 루트노드 → 가장 위에 있는 노드, 리프노드 → 자식 노드가 없는 노드
    - 깊이(레벨) → 루트노드 부터 특정 노드까지 최단 거리로 갔을 때의 거리
    - 높이 → 루트 노드부터 리프 노드까지의 거리 중 가장 긴 거리
    - 서브트리 → 트리 내의 하위 집합

### 1. 이진 탐색 트리

- 노드의 오른쪽 하위 트리 → 노드 값 보다 큰 값이 있는 노드, 왼쪽 하위 트리 → 노드 값보다 작은 값이 들어 있는 트리
- O(n)의 시간 복잡도를 가짐(선형적으로 탐색하는 case)

### 2. AVL 트리

- 이진탐색트리가 선형적인 트리가 되는 것을 방지하고 스스로 균형을 잡는 이진 탐색 트리
- 두 자식 서브트리의 높이는 항상 최대 1만큼 차이 남

### 3. 레드 블랙 트리

- 균형 이진 탐색 트리로 탐색, 삽입, 삭제 모두 시간 복잡도가 O(logn)
- 빨간색 또는 검은색의 색상을 나타내는 추가 비트를 저장하여 삽입 및 삭제 중에 트리가 균형을 유지하도록 하는 데 사용됨
- 규칙: 모든 리프, 루트노드 → 블랙, 레드 노드의 자식 노드 → 블랙

## 3. 힙

- 이진 트리 기반의 자료 구조

### 1. 최대 힙

- 루트 노드에 있는 key는 모든 자식에 있는 key 중 가장 커야 함
- 각 노드의 자식 노드와의 관계도 이와 같은 특징이 재귀적으로 이루어져야 함

### 2. 최소 힙

- 루트 노드에 있는 key는 모든 자식에 있는 key 중 최솟값 이어야 함
- 각 노드의 자식 노드와의 관계도 이와 같은 특징이 재귀적으로 이루어져야 함
- 힙에 요소가 들어오면 → 새로운 노드를 힙의 마지막 노드에 이어서 삽입

## 4. 우선순위 큐

- 대기열에서 우선순위가 높은 요소가 우선순위가 낮은 요소보다 먼저 제공되는 자료구조
- heap 기반으로 구현 됨(Deque)

## 5. MAP

- 특정 순서에 따라 key, value가 매핑된 조합으로 형성된 자료 구조
- Map<Key, Value> 형태로 구현
- 정렬을 보장하는 unordered_map과 정렬을 보장하는 map 두 가지가 있음

## 6. Set

- 특정 순서에 따라 고유한 요소를 저장하는 컨테이너 → 중복된 요소 저장 x

## 7. HashTable

- 무한에 가까운 데이터들을 유한한 개수의 해시 값으로 매핑한 테이블
- 평균 시간 복잡도 O(1)
